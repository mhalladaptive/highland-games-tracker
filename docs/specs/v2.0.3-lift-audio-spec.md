# Stone & Standard v2.0.3 — lift celebration audio sequence

Small follow-up patch on top of v2.0.2's audio wiring. v2.0.2 left
lift PR and lift Goal cards intentionally silent — "voice clips don't
land in the gym context." Oak then recorded two **environmental SFX**
clips (`barbell-drop.wav` and `loading-the-bar.wav`) that sidestep the
voice-feels-wrong problem by using the literal sound of the moment.
v2.0.3 wires them in as a four-clip sequence on every lift card.

## Risk overlay

**Normal** — UI/audio playback only, no data layer touch. Adds one
new helper (`playClipSequence`) and extends `pickCelebrationVoice`'s
return shape to accept arrays. Codex review on, one pass.

## Design decisions locked

### The lift sequence

Both **Lift PR** and **Lift Goal** cards play the same audio sequence
on save:

```
[loading-the-bar]  1.0s gap  [loading-the-bar]  1.0s gap  [loading-the-bar]  1.9s gap  [barbell-drop]
     1.5s              1.0       1.5s              1.0       1.5s              1.9        1.8s
```

Total ~8.7 seconds per lift card audio.

The narrative: loading three plates onto the bar (the warmups
adding up), pausing to settle for the lift, dropping the bar after
completing it. The 1.9s settle-before-drop captures the moment of
"ready," and the drop captures completion.

### Queue policy unchanged

The v2.0.2 multi-card queue policy (first card fires + Awesome Day
always; middle cards silent; replay stays silent) carries forward
unchanged. A lift card fires its sequence ONLY if it's the first
card in the queue. Subsequent lift cards stay silent. The Awesome
Day capstone still fires `plan-comes-together.wav` at the end of
the queue regardless of card-type history.

### Same sequence for both lift PR and lift Goal

Per Oak's call. Future v2.x can split if a clearer narrative
distinction surfaces, but the current set has the two clips Oak
recorded and a single sequence is the cleanest shape.

## What changes in `session.js`

### New voice constants

After the existing voice constants block, add:

```js
// Lift celebration sequence — fires on Lift PR and Lift Goal cards.
// Five timing units: load → 1s gap → load → 1s gap → load → 1.9s gap →
// drop. The 1.9s gap captures the lifter's settle-before-lift; the
// final drop captures completion. Total sequence ~8.7s.
const LIFT_LOADING_VOICE = 'audio/loading-the-bar.wav';
const LIFT_DROP_VOICE = 'audio/barbell-drop.wav';

  { src: LIFT_LOADING_VOICE, gapMsAfter: 1000 },
  { src: LIFT_LOADING_VOICE, gapMsAfter: 1000 },
  { src: LIFT_LOADING_VOICE, gapMsAfter: 1900 },
  { src: LIFT_DROP_VOICE,    gapMsAfter: 0    },
];
```

### Update `pickCelebrationVoice` to return the sequence for lift events

The branch order change: lift category now returns the sequence
array INSTEAD of `null`. All other branches unchanged.

```js

  if (!milestone) return null;
  if (milestone.type === 'awesomeDay') return AWESOME_DAY_VOICE;
  const event = ITEMS.find((i) => i.id === milestone.event);
  if (!event) return null;
- if (event.category !== 'throw') return null;
+ if (event.category === 'lift') return LIFT_CELEBRATION_SEQUENCE;
+ if (event.category !== 'throw') return null;
  if (event.measurementType === 'height') return HEIGHT_EVENT_VOICE;
  if (milestone.type === 'pr') {
    return THROWS_PR_VOICES[Math.floor(Math.random() * THROWS_PR_VOICES.length)];

```

The defensive fall-through for unknown category (`!== 'throw'` after
the lift branch) preserves "unknown event returns null" safety.

### Add a sequence playback helper

After `playCelebrationSound`, add a recursive helper that plays an
array of `{src, gapMsAfter}` entries in order with the specified
gaps between them. Same private-mode + autoplay tolerance pattern
as `playCelebrationSound`:

```js
// Play a sequence of clips with per-clip post-gaps. Each entry is
// {src, gapMsAfter}; gapMsAfter is the delay (ms) between this
// clip's end and the next clip's start. Same defensive try/catch
// + .catch(() => {}) pattern as playCelebrationSound — a missing
// file, undecodable clip, or blocked autoplay aborts the sequence
// silently without throwing.
function playClipSequence(sequence, index) {
  if (!Array.isArray(sequence) || index >= sequence.length) return;
  const { src, gapMsAfter } = sequence[index];
  try {
    const audio = new Audio(src);
    audio.addEventListener('ended', () => {
      const nextIndex = index + 1;
      if (nextIndex >= sequence.length) return;
      setTimeout(() => playClipSequence(sequence, nextIndex), gapMsAfter || 0);
    });
    const p = audio.play();
    if (p && typeof p.catch === 'function') p.catch(() => {});
  } catch {
    // never let audio throw into the celebration
  }
}
```

### Extend the queue-renderCard playback dispatch

`showCelebrationQueue`'s inner `renderCard` currently calls
`playCelebrationSound(src)` directly. Update to dispatch on the
picker's return type — string for single-clip slots,
array-of-sequence-entries for the lift sequence:

```js
  function renderCard() {
    slot.innerHTML = '';
    const card = buildCelebrationCard(queue[index], session, data, fullList);
    if (card) slot.appendChild(card);
    if (fireSounds) {
      const milestone = queue[index];
      const isFirst = index === 0;
      const isAwesomeDay = milestone && milestone.type === 'awesomeDay';
      if (isFirst || isAwesomeDay) {
        const voice = pickCelebrationVoice(milestone);
-       if (voice) playCelebrationSound(voice);
+       if (typeof voice === 'string') {
+         playCelebrationSound(voice);
+       } else if (Array.isArray(voice)) {
+         if (isSoundOn()) playClipSequence(voice, 0);
+       }
      }
    }
  }
```

The `isSoundOn()` check before `playClipSequence` mirrors the
existing gate inside `playCelebrationSound` so the master toggle
wins for sequence playback too.

### Manual smoke test additions

The 8.7-second sequence with timing is hard to test in
`tests.html` without mocking `Audio()` and `setTimeout`. Manual
items added to the smoke-test checklist:

- Log a session that hits only a Lift PR (e.g., Back Squat 1RM at
  a new high). Save. Verify the 3-loadings-plus-drop sequence fires
  with the correct timing pattern.
- Log a session that hits only a Lift Goal. Save. Same sequence.
- Log a Lift PR + Awesome Day. Verify the lift sequence fires
  on the first card, silence on subsequent cards, then
  `plan-comes-together.wav` on the Awesome Day card.
- Log a Throws PR + Lift PR (e.g., breaking PRs on both Open Stone
  and Back Squat in one session). Verify the throws PR voice fires
  on the first card; the lift PR card stays silent because it's
  not the first card.
