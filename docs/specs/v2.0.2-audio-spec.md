# Stone & Standard v2.0.2 — audio wiring

Patch release closing the Stage 6a audio plumbing gap. v2.0 shipped
the audio toggle and the `playCelebrationSound` helper as silent
placeholders; v2.0.2 wires the five real voice clips Oak recorded
on 2026-05-28 into the celebration queue under a per-event-type
mapping with a focused playback policy.

## Risk overlay

**Normal** — no data layer touch, no schema change. The `audio/`
folder content and `session.js` celebration-queue wiring are the
only surfaces modified. Codex review on, one pass.

## Design decisions locked

The mapping, from Oak's design call:

- **Throws PR** → randomly pick one of [`big-throw.wav`,
  `see-it-send-it.wav`]
- **Throws Goal** → `practicing.wav`
- **Height-event PR or Goal** (events with `measurementType:
  'height'` — Sheaf Toss, Weight for Height) → `up-and-over.wav`.
  Overrides the throws PR / Goal mapping when the event is a
  height type.
- **Awesome Day** → `plan-comes-together.wav`
- **Lift PR / Lift Goal** → no voice (silent).

**Multi-card queue policy**: only the FIRST card in the queue fires
its voice; subsequent cards stay silent; the Awesome Day capstone
ALWAYS fires its voice at the end of the queue (because it's always
the last card when present, and the "first card" rule already
covers it when it's the only card).

**Replay behavior**: save-only. Replay via "View Celebrations" on
Past Sessions stays silent. Same pattern as chain prompts (replay
disables them).

**Data structure**: simple per-purpose constants. No generalized
array-per-slot future-proofing — Oak's call. The throws PR slot is
the only one that needs an array today (two clips); the others are
single strings.

### Replace the `CELEBRATION_AUDIO` map

`session.js:894-898` today:

Replace with the four purpose-named constants:

```js
const SOUND_PREF_KEY = 'stone-and-standard-sound';

// Voice clip wiring. The throws PR slot rotates between two clips
// for variety; the other three slots are single-clip today. The
// shape is intentionally minimal — no generalized array-per-slot
// scaffolding. If a future slot grows to multiple clips, promote
// it to an array at that point.
const THROWS_PR_VOICES = ['audio/big-throw.wav', 'audio/see-it-send-it.wav'];
const THROWS_GOAL_VOICE = 'audio/practicing.wav';
const HEIGHT_EVENT_VOICE = 'audio/up-and-over.wav';
const AWESOME_DAY_VOICE = 'audio/plan-comes-together.wav';
```

### Add the picker

Insert immediately after the constants, before `isSoundOn`:

```js
// Resolve a milestone to its voice clip src per the v2.0.2 mapping.
// Returns null when no voice should play (lifts get no voice; an
// unknown event id is treated as no-voice for safety).
//
// Height-event override: any celebration on a height-measurement
// event (Sheaf Toss, Weight for Height) plays the height voice
// regardless of whether it's a PR or Goal milestone — the event
// type is the loudest signal.
function pickCelebrationVoice(milestone) {
  if (!milestone) return null;
  if (milestone.type === 'awesomeDay') return AWESOME_DAY_VOICE;
  const event = ITEMS.find((i) => i.id === milestone.event);
  if (!event) return null;
  if (event.category !== 'throw') return null;
  if (event.measurementType === 'height') return HEIGHT_EVENT_VOICE;
  if (milestone.type === 'pr') {
    return THROWS_PR_VOICES[Math.floor(Math.random() * THROWS_PR_VOICES.length)];
  }
  if (milestone.type === 'goal') return THROWS_GOAL_VOICE;
  return null;
}
```

### Remove the inline play call from the card builder

`session.js:1076` today plays the shout inside `buildThrowsPrCard`:

```js
  playCelebrationSound(CELEBRATION_AUDIO.shout);
  return card;
}
```

That coupling means every card build fires sound — including
replay rebuilds. Per the save-only design, sound is a queue-level
concern, not a card-build concern. Delete the `playCelebrationSound`
line (the comment above it about audio plumbing can also go).

### Wire the queue-level policy into `showCelebrationQueue`

The function starts at `session.js:1245`. Two changes:

**Read the new opt at the top of the function**:

```js
function showCelebrationQueue(session, data, options) {
  const opts = options || {};
  const fullList = Array.isArray(session && session.milestones) ? session.milestones : [];
  const queue = Array.isArray(opts.subset) ? opts.subset : fullList;
  if (queue.length === 0) return;
+ const fireSounds = !!opts.fireSounds;
```

**Fire voice when each card renders, gated on `fireSounds` AND
the per-card-position rules**:

The `renderCard` function inside `showCelebrationQueue` is the
right hook — it runs each time the queue advances. Modify:

```js
  function renderCard() {
    slot.innerHTML = '';
    const card = buildCelebrationCard(queue[index], session, data, fullList);
    if (card) slot.appendChild(card);
+   // Voice policy: first card fires; middle cards silent; the
+   // Awesome Day capstone always fires when reached. Replay
+   // (fireSounds false) stays silent throughout.
+   if (fireSounds) {
+     const milestone = queue[index];
+     const isFirst = index === 0;
+     const isAwesomeDay = milestone && milestone.type === 'awesomeDay';
+     if (isFirst || isAwesomeDay) {
+       const src = pickCelebrationVoice(milestone);
+       if (src) playCelebrationSound(src);
+     }
+   }
  }
```

The first-card-OR-AwesomeDay test correctly handles all queue
shapes:

- Single PR card (no AwesomeDay) → first card fires
- Single AwesomeDay (rare but possible from edits) → it's first
  AND it's awesomeDay; fires once
- 5-card full queue (PR-PR-Goal-Goal-AwesomeDay) → first card fires,
  middle three silent, AwesomeDay fires at end
- Queue starting with non-throws (lift PR) → first card's
  pickCelebrationVoice returns null; nothing plays at start;
  AwesomeDay at end still fires its voice

## Tests in `tests.js`

Six new picker tests. All pure-function, no DOM required:

```js
test('pickCelebrationVoice: throws PR returns one of the two throws PR voices', () => {
  for (let i = 0; i < 20; i++) {
    const src = pickCelebrationVoice({ type: 'pr', event: 'open-stone' });
    assertTrue(src === 'audio/big-throw.wav' || src === 'audio/see-it-send-it.wav');
  }
});

test('pickCelebrationVoice: throws Goal returns practicing.wav', () => {
  assertEqual(pickCelebrationVoice({ type: 'goal', event: 'open-stone' }), 'audio/practicing.wav');
});

test('pickCelebrationVoice: height-event PR returns up-and-over.wav (overrides throws PR mapping)', () => {
  assertEqual(pickCelebrationVoice({ type: 'pr', event: 'sheaf-toss' }), 'audio/up-and-over.wav');
  assertEqual(pickCelebrationVoice({ type: 'pr', event: 'weight-over-bar' }), 'audio/up-and-over.wav');
});

test('pickCelebrationVoice: height-event Goal returns up-and-over.wav (overrides throws Goal mapping)', () => {
  assertEqual(pickCelebrationVoice({ type: 'goal', event: 'sheaf-toss' }), 'audio/up-and-over.wav');
});

test('pickCelebrationVoice: lift PR returns null (lifts have no voice)', () => {
  assertEqual(pickCelebrationVoice({ type: 'pr', event: 'deadlift' }), null);
});

test('pickCelebrationVoice: lift Goal returns null', () => {
  assertEqual(pickCelebrationVoice({ type: 'goal', event: 'deadlift' }), null);
});

test('pickCelebrationVoice: awesomeDay returns plan-comes-together.wav', () => {
  assertEqual(pickCelebrationVoice({ type: 'awesomeDay' }), 'audio/plan-comes-together.wav');
});

test('pickCelebrationVoice: unknown event id returns null safely', () => {
  assertEqual(pickCelebrationVoice({ type: 'pr', event: 'not-a-real-event' }), null);
});
```
