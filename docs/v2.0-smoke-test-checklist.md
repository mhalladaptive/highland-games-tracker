# Stone & Standard — v2.0 cross-device smoke test checklist

The gating check before tagging `v2.0.0`. One seeded scenario, walked
end-to-end on **desktop + iPhone Safari**, exercising the surface of
all twelve shipped stages in user-flow order.

Designed for one sitting — ~30–45 minutes. Run desktop and iPhone in
parallel where it makes sense (same scenario both sides), or back-to-
back. Pass criteria at the bottom.

---

## Pre-flight

Have these ready before you start:

- The local dev server running. From the repo root:

  ```
  cd ~/dev/highland-games-tracker
  python3 -m http.server 8000
  ```

- Your Mac's LAN IP for the iPhone load — `ipconfig getifaddr en0`
  (Wi-Fi) usually answers. Note it here for the run: `___.___.___.__`

- A cache-bust query string with the current timestamp. Use the format
  `?v=YYYYMMDD-HHMM`. Pick one timestamp at run start and reuse it for
  every URL in this checklist. Note it here: `?v=___________`

  *Lesson banked 2026-05-28: iOS Safari caches localhost CSS/JS
  aggressively. The desktop Cmd+Shift+R trick does not translate.
  Cache-bust query string is the first mobile-load step, not a
  fallback.*

- A saved v1 `comeback-tracker` backup JSON if you have one, for the
  optional import-compat test at the end. If not, that step is "skip
  with note."

URLs for the run (fill in once and reuse):

- Desktop (Private Window, Cmd+Shift+N):
  `http://localhost:8000?v=___________`
- iPhone (Private tab):
  `http://___.___.___.__:8000?v=___________`

## Step 1 — First load and cache discipline

**Do:** Open both URLs above. Desktop in a Private Window (Cmd+Shift+N),
iPhone in a Private tab.

*Lesson banked 2026-05-28: a Claude-driven browser MCP tab does NOT
give a clean-room localStorage — it shares storage with your normal
Chrome at the same origin. Clean-room private browsing is the
human-driven path.*

**Verify:**
- [ ] Desktop: page loads, app surface is **Stone & Standard** branded
      (header, page title, README link if surfaced).
- [ ] iPhone: page loads, same brand surface.
- [ ] Neither device shows "Highland Games Tracker" anywhere on screen.
- [ ] No console errors on desktop (open DevTools briefly to check).

*Cache-bust caveat banked 2026-05-28: clicking nav-bar links between
pages (Set PRs & Goals / Log Session / Progress / Tests) does NOT
carry the `?v=` query string forward — each page is its own load.
On iPhone Safari this means cache-bust only protects `index.html`'s
initial bundle. If a different page shows stale behavior, reload
that page directly with its own `?v=` query string before
troubleshooting further.*

## Step 5 — Celebration card queue (Stage 4b + 6c)

This is the highest-stakes step of the smoke test — the ornate 6c
template renders here.

**Card queue should fire in this order** (verified 2026-05-28: the
queue groups all PR cards first then all Goal cards, not throws-then-
S&C as initially assumed — see `session.js` celebration ordering):

1. **Open Stone PR** card — **ornate template** (PR shield + parchment
   + stone plaque + brass scroll + thistle/laurel framing + misty
   mountain backdrop, with Open Stone silhouette in the medallion).
2. **Back Squat PR** card — simpler v2.0 path.
3. **Open Stone Goal** card — simpler v2.0 path.
4. **Back Squat Goal** card — simpler v2.0 path.
5. **Awesome Day** capstone — lists all four milestones above.

*Queue UX note banked 2026-05-28: advance is by tap on the overlay
or Enter / Space / Right Arrow. The × button ends the queue early —
it is NOT a "next card" button.*

**Verify the ornate card (1) specifically:**
- [ ] Desktop: silhouette renders inside the parchment medallion with
      clean alpha (no white rectangle behind it).
- [ ] iPhone: same.
- [ ] Plaque body text reads cleanly against the dark stone — that's
      the `#fff8d8` pale-warm-cream fix from the 2026-05-28 contrast

      light, flag it.
- [ ] Brass scroll metadata (date / games / location / wordmark) reads
      cleanly.
- [ ] Wordmark says "Stone & Standard," not "Highland Games Tracker."

**Verify the queue ordering and chain:**
- [ ] PR cards fire before Goal cards.
- [ ] Throws cards fire before S&C cards.
- [ ] Awesome Day fires last and lists all four prior milestones.
- [ ] Audio toggle visible on the overlay, defaults to off.
- [ ] On Goal card dismiss, the **chain prompt** appears: *"Want to
      set a new goal for [event]?"*
- [ ] On the Open Stone Goal chain prompt, tap **set a new goal** and
      enter `35`. Confirm the new goal sticks.

## Step 9 — Backup export and re-import

Self-contained round-trip: export the smoke-test data, clear, re-import.

**Do, both devices:**
- Navigate to **Set PRs & Goals** and scroll to the bottom — the
  "Data" section sits there with **Download Backup** and **Restore
  from Backup** buttons. (Verified 2026-05-28: backup affordance lives
  on the Set page, not on Profile / Settings.)
- **Download Backup** to export the JSON.
- Open the downloaded file in a text editor (desktop only). Envelope
  shape per `app.js:461`:
  ```
  {
    "appName": "stone-and-standard",
    "exportedAt": "<ISO timestamp>",
    "data": { "version": 2, ... }
  }
  ```
  - [ ] `appName` reads `stone-and-standard`.
  - [ ] `exportedAt` is a recent ISO timestamp.
  - [ ] `data.version` is `2`. (Note: there is NO top-level `version`
        field on the envelope — version sits inside `data`.)
  - [ ] Filename matches `stone-and-standard-backup-YYYY-MM-DD.json`.
- **Restore from Backup** with the file you just exported.

**Verify:**
- [ ] Re-import succeeds; data still intact (Open Stone 33/35,
      Back Squat 250/225 with soft callout).

**Optional — v1 compat check (if you have a v1 backup):**
- Import a saved `comeback-tracker` v1 backup JSON.
- [ ] Validation accepts the legacy `appName`.
- [ ] Schema migration runs (baselines → prs, etc.).
- [ ] Hard-coded v1 lifts arrive as `userLifts` entries with their
      names + protocols preserved.
