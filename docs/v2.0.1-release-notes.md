# Stone & Standard v2.0.1

First-hour polish from the v2.0 launch. Small patch release —
nothing here changes how the app works, just makes a few rough
edges sit right.

## What's new

### Edit your profile, any time

If you dismissed the profile setup before finishing on first launch
(easy to do accidentally), there was no way back to complete it.
There is now. A small **⚙ Edit profile** button at the top of the
Set PRs & Goals page opens the profile form pre-populated with your
current values. Update anything, save, you're done.

The button is there for the rest of the use cases too — aging up
into Masters, switching weight schedule, fixing a typo in your
name. Profile lifecycle is a real thing; the app now respects it.

### PR celebration card — composition polish

A few small visual fixes to the ornate throws PR card surfaced
during launch testing on real devices:

- Plaque text and brass scroll text both sit centered within
  their illustrated regions now (each was a fingernail too high).
- The date on the brass scroll banner now **curves along the
  scroll's natural arc**, matching the headline's weight and
  reading as engraved-into-the-scroll typography rather than text
  floating over the illustration.
- The Stone & Standard wordmark moved to a **bottom-left
  book-spine position** — rotated vertical, reads bottom-to-top —
  so it sits as decorative attribution rather than competing with
  the card's headline mark.
- The plaque headline now reads **"NEW PR ACHIEVED"** in a single
  strong label instead of the earlier two-line "Personal Record /
  New PR Achieved" stack.

If you want to see the difference, log a throw that beats your
PR on any throws event and watch the card fire — or replay an
existing one from Past Sessions via **View Celebrations**.

## Behind the scenes

- Stage 4b regex coverage was updated for the headline rename;
  test suite back at green at the new count.
- Profile modal refactored to support multiple opens cleanly
  without listener accumulation.
- `setupCompletedAt` timestamp now preserved across re-edits
  (was being re-stamped on every save).

## Migration notes

Open the app — the polish is just there. Your data carries forward
untouched. No account, no signup, no manual migration; same as
always.
