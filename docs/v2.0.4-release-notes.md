# Stone & Standard v2.0.4

Sound turns on by itself now.

The celebration audio has been wired up since v2.0.2, but it
stayed off by default — you had to find the toggle on a
celebration card to flip it on. v2.0.4 swaps that posture:
sound now plays by default. The next time you save a session
that fires a celebration card, you'll hear it.

## What's new

### Default-on, opt-out instead of opt-in

If you've never touched the sound toggle, sound now plays. If
you previously turned sound off explicitly, your choice stands
— you stay silent. The only people affected are those who left
it default; everyone else's preference holds.

### A sound toggle inside the profile

The celebration overlay's toggle still works exactly as it did,
but there's now a second place to control sound: the **Profile**
modal, accessible via the **⚠ Edit profile** button at the top
of the Set PRs & Goals page.

```
[ ] Play celebration sounds
    Sound stays off if you turn it off here. The overlay
    on a celebration card has the same toggle.
```

Both access points read and write the same preference. Toggle
it off in the profile, the overlay reflects that. Toggle it on
from a celebration card, the profile's checkbox shows checked
next time you open it. They stay in sync automatically.

### Save-immediately

The Profile modal's sound checkbox doesn't wait for the Save
button — flip it, and the preference saves on the spot. The
Save button stays focused on the rest of your profile (name,
gender, class, tier).

## Migration notes

Open the app on a device where you've never toggled sound, hit
your next PR — and there's the voice. Open the profile to find
the new Sound row at the bottom of the form. That's it.

No data touch, no schema change. The sound preference still
lives where it always has (a small standalone localStorage key);
this release just changes how it defaults and adds a second way
to control it.

## Thanks

To the launch-week feedback loop hitting hard enough that audio
